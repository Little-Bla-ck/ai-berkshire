# 短线投资：事件、趋势与风控优先的交易计划

对 `$ARGUMENTS` 执行短线投资研究。目标不是证明一家公司值得长期持有，而是在有限时间窗口内回答：

> 这笔短线交易的催化剂是什么？市场是否已经反映？进场、失效、止损、目标、仓位如何设置？

本 skill 仅用于学习与研究，不构成投资建议；不得替用户下单或承诺收益。

## 适用边界

默认时间窗口：

| 类型 | 持有周期 | 核心问题 |
|---|---:|---|
| 隔夜/事件跟随 | 1-5 个交易日 | 催化剂是否仍在发酵，是否有二次确认？ |
| 波段交易 | 1-4 周 | 趋势、资金、估值/预期差是否共振？ |
| 中短线主题 | 2-8 周 | 行业叙事是否扩散，龙头是否确认？ |

不做：
- 不做高频、盘口抢单、秒级/分钟级策略。
- 不把单一技术图形当成买入理由。
- 不在无法确认最新价格、成交量和事件来源时给出确定性结论。
- 不为了短线机会忽略流动性、财务造假、退市、监管和停牌风险。

## 输入参数

先从用户输入中提取：

| 参数 | 说明 | 默认 |
|---|---|---|
| 标的 | 公司名、股票代码、ETF、指数均可 | 必填 |
| 市场 | A 股、港股、美股、ETF、加密等 | 未给出则先识别 |
| 交易方向 | 做多、做空、观望 | 默认只评估做多；做空需明确 |
| 时间窗口 | 1 日、1 周、1 月等 | 默认 2-4 周 |
| 用户成本/仓位 | 已持仓时用于判断加减仓 | 未给出则只做新计划 |
| 触发背景 | 新闻、财报、股价异动、主题、技术突破 | 未给出则先做快速体检 |

如缺少标的或市场，先询问；其他参数可用默认值并在报告中说明。

## 核心原则

1. **先证伪，再下注**：短线计划必须写清“什么发生就说明我错了”。
2. **催化剂优先**：没有催化剂的短线，只能按技术反弹或趋势跟随处理，置信度自动下调。
3. **风险收益比先于胜率**：最低要求预期上行空间 / 下行风险 >= 2:1；达不到则建议观望。
4. **流动性是硬门槛**：成交额、换手、买卖价差不足时，不给积极建议。
5. **仓位由止损距离决定**：先定最大亏损，再反推仓位；禁止因“看好”而放大仓位。
6. **短线不等于无基本面**：财报质量、估值极端、监管/退市风险会改变止损和仓位上限。

## 执行流程

### 第一步：快速定位交易类型

将标的归入一种主类型，并说明理由：

| 类型 | 识别信号 | 主要验证 |
|---|---|---|
| 事件驱动 | 财报、订单、政策、并购、回购、诉讼、产品发布 | 事件是否真实、影响是否超预期 |
| 趋势延续 | 放量突破、均线多头、相对强度领先 | 量价是否健康、是否过热 |
| 超跌反弹 | 连续下跌、利空落地、估值/情绪极端 | 是否止跌、是否有催化剂修复 |
| 主题轮动 | 行业政策、产业趋势、资金抱团 | 主题是否扩散，谁是强度龙头 |
| 财报交易 | 业绩/指引/电话会超预期或低于预期 | 预期差、卖方修正、管理层措辞 |

如果无法归类，输出“无清晰短线交易假设”，并降低建议强度。

### 第二步：复用现有能力收集事实

按场景调用/参考仓库现有 skill：

- 股价异动、新闻归因：优先使用 `news-pulse` 的四维侦察框架（公司事件、监管政策、行业对手、市场情绪）。
- 财务和估值底线：参考 `financial-data`，关键财务数据至少两源交叉验证。
- 质量排雷：参考 `quality-screen`，快速检查现金流、债务、商誉、客户集中、监管/退市等风险。
- 已有持仓：参考 `thesis-tracker`，判断短线动作是否违反原始长期论文。
- 深度研究缺口：若短线催化剂触发长期假设重审，建议转入 `investment-research` 或 `investment-team`。

必须获取并标注日期/来源：

1. 最新价格、涨跌幅、成交额、换手率或成交量。
2. 近 5、20、60 个交易日表现，和行业/指数对比。
3. 最新关键新闻、公告、财报、政策或主题事件。
4. 机构预期变化、资金流向、做空数据或龙虎榜/大宗交易（按市场可得性）。
5. 关键基本面风险：盈利质量、现金债务、监管、停牌/退市、诉讼。

### 第三步：催化剂强度评分

给每个候选催化剂打分：

| 维度 | 0 分 | 1 分 | 2 分 |
|---|---|---|---|
| 真实性 | 传闻/未证实 | 二手媒体 | 官方公告/一手数据 |
| 超预期 | 市场已充分预期 | 略超预期 | 明显超预期 |
| 持续性 | 一次性噪音 | 影响 1-2 周 | 影响一个季度以上 |
| 传播性 | 小范围关注 | 行业内关注 | 全市场主题扩散 |
| 可验证性 | 难以跟踪 | 有间接指标 | 有明确后续验证点 |

总分解释：
- 8-10 分：强催化剂，可进入交易计划。
- 5-7 分：中等催化剂，需要技术/资金面共振。
- 0-4 分：弱催化剂，默认观望或只做极小仓位试错。

### 第四步：技术位置与资金面

短线技术分析只回答“赔率、拥挤度、失效点”，不要画玄学结论。

必须检查：

| 模块 | 要看什么 | 输出 |
|---|---|---|
| 趋势 | 20/60 日均线、阶段高低点、相对强度 | 上升/震荡/下降 |
| 成交量 | 突破是否放量、回调是否缩量 | 健康/背离/异常 |
| 关键价位 | 近期平台、缺口、前高前低、密集成交区 | 支撑/阻力/失效点 |
| 波动率 | ATR、近期振幅、财报/事件风险 | 止损距离 |
| 资金面 | 主力资金、北向/南向、ETF 流入、做空变化 | 顺风/逆风/不明 |

若没有可靠行情数据，明确写“技术位置无法验证”，不得给精确买点。

### 第五步：风险收益与仓位计算

建立 3 个价格点：

| 项目 | 含义 |
|---|---|
| 入场区间 | 计划买入/加仓价格区间，不追高到无赔率 |
| 止损/失效价 | 跌破或事件证伪后退出的位置 |
| 目标区间 | 第一目标和第二目标，来自阻力位、估值修复或事件影响测算 |

计算：

```text
每股风险 = 入场价 - 止损价
每股潜在收益 = 目标价 - 入场价
风险收益比 = 每股潜在收益 / 每股风险
最大可亏损金额 = 账户权益 * 单笔风险上限
建议仓位股数 = 最大可亏损金额 / 每股风险
```

仓位规则：

| 信号质量 | 单笔风险上限 | 仓位建议 |
|---|---:|---|
| 强催化剂 + 技术确认 + 流动性好 | 账户 0.8%-1.0% | 可分批建仓 |
| 中等催化剂或技术未确认 | 账户 0.3%-0.5% | 小仓试错 |
| 弱催化剂/流动性差/数据缺口大 | 0%-0.2% | 观望优先 |

如用户未提供账户权益，只输出“按每亏损 1 元可承受风险反推”的通用仓位公式，不虚构账户规模。

### 第六步：交易计划

用条件句写计划，不写口号：

| 场景 | 行动 |
|---|---|
| 突破确认 | 若价格放量站上关键位并维持，则按计划买入/加仓 |
| 回踩确认 | 若回踩支撑缩量不破，则分批买入 |
| 追高限制 | 若开盘/盘中高开超过合理入场区间，则放弃或等待回落 |
| 失效退出 | 若跌破止损价、催化剂被证伪、成交量异常放大下跌，则退出 |
| 止盈 | 到第一目标减仓，剩余仓位用移动止损跟踪 |
| 时间止损 | 到计划窗口仍未兑现催化剂，则退出或重审 |

必须给出“今日/本周最该等的信号”，避免把报告写成模糊观点。

## 最终判断标准

结论必须落在以下之一：

| 结论 | 使用条件 |
|---|---|
| 观望 | 无清晰催化剂、赔率不足、数据缺口大、流动性不足 |
| 小仓试错 | 催化剂中等，赔率达标，但仍需技术确认 |
| 分批买入 | 催化剂强、技术确认、风险收益比达标 |
| 持有不加仓 | 已有仓位但新买赔率不足 |
| 减仓 | 目标接近、拥挤过热、催化剂兑现 |
| 退出 | 失效条件触发或原假设被证伪 |

## 复盘要求

每个短线计划都必须设置复盘点：

- 价格复盘：触及止损、第一目标、第二目标时。
- 时间复盘：到达计划窗口一半时；到期仍未兑现时。
- 事件复盘：公告、财报、电话会、政策、竞品事件落地后。

复盘时回答：
1. 原催化剂是否兑现？
2. 市场反应是否弱于/强于预期？
3. 技术结构是改善还是恶化？
4. 是否需要转入 `news-pulse`、`earnings-review` 或 `thesis-tracker`？
5. 当前动作是继续、减仓、退出，还是重建计划？

## 红线

出现以下任一情况，默认不做或立即重审：

- 最新价格、成交量或关键事件无法验证。
- 跌破止损后用“长期看好”替代短线纪律。
- 催化剂只是传闻，且无法找到独立来源。
- 风险收益比低于 2:1，却因为情绪想追。
- 单笔亏损可能超过账户预设风险上限。
- 标的存在停牌、退市、财务造假、流动性枯竭或重大监管风险。
